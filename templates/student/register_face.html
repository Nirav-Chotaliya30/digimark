How Face Registration Works
  1. Webcam captures 5 frames (1-second gap).
  2. Frames are sent to `POST /api/register-face` as base64 images.
  3. Backend extracts a 128-d face encoding for each frame.
  4. System performs liveness check using face movement.
  5. Average encoding is stored for your student record.
Where It Is Stored

Database: SQLite (`digimark.db`)

Table: `students`

Columns: `face_encoding` (JSON vector), `face_registered` (boolean)
